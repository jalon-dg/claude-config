# 采访问题库 v2.0

> **版本号**：v2.0.0
> **最后更新**：2026-02-07
> **核心升级**：四轮灵魂拷问 + 动态追问机制

---

## 采访理念

**核心转变**：从"信息收集"转变为"故事挖掘"

- 让用户在回答中发现自己都没意识到的洞见
- 像朋友聊天，不像记者采访
- 追问比预设问题更重要
- 挖掘故事和情感，而非罗列事实

---

## 采访结构：四轮灵魂拷问

| 轮次 | 名称 | 目的 | 必选/可选 |
|-----|------|------|----------|
| 第一轮 | 破冰与定位 | 建立信任，了解身份/动机/读者 | 必选 |
| 第二轮 | 深挖与具象化 | 挖掘具体故事、细节、数据 | 必选 |
| 第三轮 | 灵魂拷问 | 触及本质，挖掘独特洞见 | 必选 |
| 第四轮 | 验证与补充 | 确认理解，填补空白 | 可选 |

**最少轮次保障**：无论如何至少完成前3轮。

---

## 第一轮：破冰与定位（必选）

**目的**：建立信任，了解用户身份和写作动机

**核心问题**：你是谁？为什么关注这个？想写给谁看？

1. **身份定位**：你平时主要做什么工作？和这个主题有什么关系？
2. **接触契机**：你是怎么知道/接触到这个东西的？（朋友推荐、社交媒体刷到、工作需要？）
3. **写作动机**：为什么想写这篇文章？是有什么触动你的点吗？
4. **读者画像**：你希望谁来读这篇文章？他们最关心什么？

**本轮目标**：
- [ ] 了解用户身份背景
- [ ] 明确写作动机
- [ ] 确定目标读者

---

## 第二轮：深挖与具象化（必选）

**目的**：挖掘具体故事、细节、数据

**核心问题**：能举个具体例子吗？当时的细节是什么？

1. **具体场景**：能描述一个你使用/体验它的具体场景吗？当时在做什么？
2. **关键细节**：那个场景里，有什么细节让你印象特别深？
3. **第一印象**：第一次接触它时，最让你惊讶/困惑/兴奋的是什么？
4. **对比体验**：和你之前用过的类似东西比，最大的不同是什么？
5. **数据支撑**：有没有什么数据或事实可以支撑你的观点？

**本轮目标**：
- [ ] 获得2-3个具体案例/故事
- [ ] 挖掘到生动细节
- [ ] 有对比或数据支撑

---

## 第三轮：灵魂拷问（必选）

**目的**：触及本质，挖掘独特洞见

**核心问题**：你觉得本质是什么？如果只能说一句话？

1. **本质追问**：抛开所有细节，你觉得它的本质是什么？
2. **一句话总结**：如果只能用一句话向朋友推荐/解释它，你会怎么说？
3. **反向思考**：有没有什么是大家都在说，但你觉得不对的？
4. **争议观点**：关于它，你有没有什么"不敢说"或"怕被喷"的观点？
5. **未来判断**：你觉得它会往什么方向发展？为什么这么判断？

**本轮目标**：
- [ ] 提炼出核心观点
- [ ] 挖掘独特洞见
- [ ] 发现差异化视角

---

## 第四轮：验证与补充（可选）

**目的**：确认理解，填补空白

**触发条件**：前三轮信息不够充分，或用户意犹未尽

1. **理解确认**：我整理一下你的核心观点是[...]，对吗？有没有理解偏差？
2. **遗漏检查**：有没有什么我没问到但你特别想说的？
3. **情感确认**：你希望读者看完文章后有什么感受？
4. **风险提示**：有没有什么敏感点是需要注意的？
5. **补充素材**：你手上有没有什么截图、数据、链接可以作为素材？

**本轮目标**：
- [ ] 确认理解无偏差
- [ ] 填补信息空白
- [ ] 收集补充素材

## 动态追问机制

根据用户回答自动触发追问，让采访更自然、更深入。

### 追问触发规则

| 触发条件 | 追问类型 | 示例问题 |
|---------|---------|---------|
| 回答笼统（<20字或太抽象） | 具象化追问 | "能举个具体的例子吗？""当时具体是什么情况？" |
| 有趣细节一笔带过 | 深挖追问 | "这个点很有意思，能展开说说吗？""后来呢？" |
| 表达强烈情感（惊讶/愤怒/兴奋） | 情感追问 | "当时是什么感受？""为什么会有这种感觉？" |
| 前后表述矛盾 | 澄清追问 | "前面你说...现在又说...能解释一下吗？" |
| 回答敷衍或回避 | 开放追问 | "如果换个角度想呢？""假设没有限制的话呢？" |
| 提到具体数据 | 数据追问 | "这个数据是怎么来的？""还有其他数据支撑吗？" |
| 提到他人观点 | 立场追问 | "你自己怎么看？""你同意这个说法吗？" |

### 追问原则

- **自然融入**：追问要像朋友聊天时的自然反应，不要像审讯
- **适可而止**：同一个点最多追问2次，避免让用户感到压迫
- **抓大放小**：优先追问与核心观点相关的内容
- **保持好奇**：用真诚的好奇心提问，而非机械执行

### 追问话术库

**具象化追问**：
- "能举个具体的例子吗？"
- "当时具体是什么情况？"
- "比如说？"
- "能描述一下那个场景吗？"

**深挖追问**：
- "这个点很有意思，能展开说说吗？"
- "后来呢？"
- "然后发生了什么？"
- "这里面有什么故事吗？"

**情感追问**：
- "当时是什么感受？"
- "为什么会有这种感觉？"
- "这让你想到了什么？"
- "听起来你对这个挺[兴奋/失望/惊讶]的？"

**澄清追问**：
- "我没太理解，能再解释一下吗？"
- "你的意思是...对吗？"
- "这和你前面说的...是什么关系？"

**开放追问**：
- "如果换个角度想呢？"
- "假设没有限制的话呢？"
- "你觉得理想情况应该是怎样的？"

## 采访结束判断

### 结束标准

满足以下**3项**即可结束采访：

- [ ] 有明确核心观点（用户能一句话总结）
- [ ] 有2-3个具体案例/故事（有细节、有画面感）
- [ ] 有目标读者画像（知道写给谁看）
- [ ] 有情感共鸣点（知道想传递什么情绪）
- [ ] 用户表示"差不多了"或"没什么要补充的了"

### 信息充分度检查

在结束采访前，快速检查：

| 维度 | 是否充分 | 缺失时的补救问题 |
|-----|---------|----------------|
| 核心观点 | ? | "如果只能说一句话，你最想表达什么？" |
| 具体案例 | ? | "能分享一个让你印象最深的场景吗？" |
| 读者画像 | ? | "你觉得什么样的人会对这篇文章感兴趣？" |
| 情感基调 | ? | "你希望读者看完后有什么感受？" |
| 差异化视角 | ? | "关于这个话题，你有什么不一样的看法？" |

## 采访技巧

### 基本原则

- 每次最多问 **3-4 个问题**，避免压迫感
- 用户可以口述回答，也可以提供文档路径
- 追问要自然，像朋友聊天
- 用户说"我把回答放到XXX文件里了"时，主动读取文件

### 语气示范

**好的提问方式**：
- "聊聊你第一次用它的感受？"
- "有没有什么让你特别惊喜或失望的地方？"
- "如果要给朋友推荐，你会怎么说？"

**避免的提问方式**：
- "请描述您的使用体验。"（太正式）
- "请从以下几个维度回答..."（太结构化）
- "请详细说明..."（太像问卷）

### 特殊情况处理

**用户回答很简短**：
- 不要急于进入下一个问题
- 用追问引导展开："能具体说说吗？""比如呢？"

**用户跑题**：
- 先肯定用户分享的内容
- 温和引导回来："这个很有意思，不过我更好奇的是..."

**用户表示不知道怎么回答**：
- 换个角度提问
- 给出选项引导："比如是因为A还是B？"
- 用假设性问题："如果让你选的话..."

**用户提供了文档/链接**：
- 主动读取并消化
- 基于文档内容进行追问
- 不要重复问文档里已有的信息

## 采访输出

采访结束后，用表格整理核心观点，向用户确认：

```
| 维度 | 用户观点 |
|------|----------|
| 身份背景 | ... |
| 核心观点 | ... |
| 关键案例 | ... |
| 独特洞见 | ... |
| 目标读者 | ... |
| 情感基调 | ... |
```

（表格维度根据实际采访内容灵活调整）
