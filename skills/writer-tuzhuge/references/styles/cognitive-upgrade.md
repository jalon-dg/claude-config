# 认知升级风格

融合"朋友聊天风格"的口语化与"Dan Koe 风格"的深度框架，偏向理性思辨，适合认知类、成长类、深度分析类内容。

---

## 一、风格定位

**深度来自 Dan Koe，温度来自朋友聊天，但整体偏向深度。**

- 框架构建、概念定义、结构化思考（Dan Koe 核心）
- 口语化表达、适度自嘲、保持温度（朋友聊天点缀）
- 帮助读者"认知升级"，而不只是"获取信息"

---

## 二、结构特点

### 两位数数字分章（01、02、03...）

- 用 01、02、03... 分大章节（不用罗马数字 I、II、III）
- 每章有独立的小标题，清晰表达该章主题
- 章节之间有明确的逻辑递进关系
- 关键章节开头可引用名人名言

**示例：**
```
01 – 被动消费 vs 主动参与：一个本质区别

02 – 对象恒常性：为什么"记住场景"这么难

03 – Genie 的能力架构：输入、交互、迭代
```

---

## 三、名人名言规范

### 必须使用中文表达

- 所有名人名言**必须翻译成中文**，不使用英文原文
- 作者名字使用中文译名
- 格式：引用块 + 破折号 + 作者名

**正确示例：**
```
> 预测未来的最好方式，就是去创造它。
> – 艾伦·凯
```

```
> 地图不是领土本身。
> – 阿尔弗雷德·科日布斯基
```

**错误示例（禁止）：**
```
> The best way to predict the future is to invent it.
> – Alan Kay
```

### 名言使用原则

- 不是每章都必须有名言，只在关键位置使用
- 名言必须与章节主题高度相关
- 优先选择有洞察力的名言，而非鸡汤
- 中外名人皆可，不限于西方思想家

---

## 四、论证方式

### 概念定义 + 框架构建 + 对比分析

**1. 先定义概念（什么是 X，什么不是 X）**
```
所有内容消费可以分为两种模式：

被动消费：内容是固定的，你只能接收。

主动参与：内容是动态的，你可以介入。
```

**2. 构建框架帮助理解**
```
Genie 的能力可以分为三层：

第一层：World Sketching —— 输入
第二层：World Exploration —— 交互
第三层：World Remixing —— 迭代

公式：输入 → 交互 → 迭代
```

**3. 用对比表揭示本质**
```
| 维度 | Sora | Genie |
|------|------|-------|
| 核心问题 | 如何生成逼真的视频 | 如何生成可交互的世界 |
| 用户角色 | 观众（被动） | 玩家（主动） |
```

**4. 引入专业概念增加深度**
- 可以引入认知科学、系统论、哲学等领域的概念
- 但必须用通俗语言解释清楚
- 目的是帮助理解，不是炫技

---

## 五、句式节奏

### 长短交替 + 单句成段

- 短句制造冲击力，长句展开论证
- 关键观点用单句成段强调
- 比纯 Dan Koe 更口语，但比朋友聊天更克制

**示例：**
```
这不是量变，是质变。
```

```
这是真正的无监督学习在世界模型上的应用。
```

```
两者都很厉害，但解决的问题不同。
```

---

## 六、口语化表达

### 保持温度但不过度

- "说白了"、"为什么？"、"这听起来像废话"
- 适度使用，不像朋友聊天那样密集
- 目的是让深度内容更易读，而非制造亲切感

**示例：**
```
这听起来像废话。但对 AI 来说，这是一个极其困难的问题。
```

```
说白了，它看过超过 20 万小时的视频，从中学会了"世界是怎么运转的"。
```

---

## 七、类比手法

### 结构化类比为主

- 类比要有解释力，不只是有趣
- 优先使用能揭示本质的类比
- 可以用生活化类比，但要服务于论证

**示例：**
```
被动消费像坐在电影院里，银幕放什么你看什么。

主动参与像走进一片森林，你往哪走，风景就往哪变。
```

```
Sora 像是一个顶级摄影师。你告诉它想要什么画面，它给你拍出来。但拍完就是拍完了，你不能走进照片里。

Genie 像是一个即兴的建筑师。你告诉它想要什么空间，它给你造出来。然后你可以走进去，四处看看，甚至重新装修。
```

---

## 八、自嘲与幽默

### 适度保留，点到为止

- 不像土著哥风格那样密集
- 只在合适的位置使用，通常是局限性部分
- 目的是保持人味，不是制造笑点

**示例：**
```
我自己只有 Pro 套餐，还没能亲自体验。
```

```
目前的体验大概介于"云游戏"和"幻灯片"之间。
```

---

## 九、收尾风格

### 升华思考 + 签名

- 结尾要有思辨感，不只是总结
- 可以抛出一个问题让读者思考
- 保留"我是土著哥，我们下期见"签名

**示例：**
```
我们正站在一个新时代的门槛上。

唯一的问题是：你是想当观众，还是想当玩家？

我是土著哥，我们下期见。
```

---

## 十、标志性表达

必须自然使用以下元素：

- 两位数分章：01、02、03...
- 名人名言（中文）开篇关键章节
- 框架/模型/公式总结
- 对比表格
- "说白了"、"为什么？"等口语化连接
- 专业概念 + 通俗解释
- 固定结尾："我是土著哥，我们下期见。"

---

## 十一、禁忌

### 绝对不要

- ❌ 英文名人名言——必须翻译成中文
- ❌ 罗马数字分章（I、II、III）——用 01、02、03
- ❌ "首先、其次、最后"——用数字分章代替
- ❌ 纯鸡汤式名言——要有洞察力
- ❌ 炫技式引用概念——必须解释清楚
- ❌ 过度口语化——保持深度感
- ❌ 过度自嘲——点到为止
- ❌ 没有框架的流水账——必须有结构

---

## 十二、适用场景

- 认知升级、思维方式类文章
- 深度分析、趋势解读类内容
- 技术原理科普（面向有一定基础的读者）
- 个人成长、自我提升类内容
- 需要"有深度但不枯燥"的主题

---

## 十三、核心原则

**用框架降低理解门槛，用口语保持阅读温度，用深度帮助认知升级。**

- 每篇文章必须有至少一个核心框架或模型
- 抽象概念必须具象化解释
- 不只告诉读者"是什么"，更要告诉"为什么"和"意味着什么"
- 读完之后，读者的认知应该有所提升
- 深度和可读性可以兼得
