# 文章质量评分系统 v1.0

> **版本号**：v1.0.0
> **最后更新**：2026-02-07
> **用途**：初稿审核的量化评分标准

---

## 评分概览

### 六大评分维度

| 维度 | 权重 | 说明 |
|-----|------|------|
| 语言自然度 | 25% | 去AI味核心指标 |
| 结构合理性 | 20% | 避免教科书结构 |
| 个人特色 | 20% | 观点/情绪/经历表达 |
| 句式规范 | 15% | 长句控制，节奏感 |
| 风格一致性 | 10% | 与选定风格的匹配度 |
| 可读性 | 10% | 段落长度、过渡自然度 |

### 通过标准

- **总分阈值**：≥ 6.0分（10分制加权平均）
- **硬性条件**：四项必须全部满足

---

## 硬性条件（一票否决）

无论总分多高，以下四项必须全部满足：

| 条件 | 标准 | 检测方法 |
|-----|------|---------|
| 禁止词控制 | ≤ 2个 | 按 anti-ai-rules.md 禁止词清单检测 |
| 长句控制 | 超30字长句 ≤ 3个 | 统计每句字数 |
| 个人表达 | ≥ 3处 | 检测"我觉得""我认为""说实话"等主观表达 |
| 结构词控制 | 无连续"首先、其次、最后" | 全文检索 |

## 评分标准详解

### 1. 语言自然度（25%）

**核心指标**：去AI味程度

| 分数 | 标准 | 典型表现 |
|-----|------|---------|
| 9-10 | 完全无AI味 | 读起来像真人写的，有个人语言习惯 |
| 7-8 | 偶有套话 | 整体自然，有1-2处可优化 |
| 5-6 | 需要修改 | 有3-5处AI味表达 |
| 3-4 | AI味明显 | 多处禁止词，套话连篇 |
| 0-2 | 需要重写 | 严重AI味，像机器生成 |

**检测要点**：
- 禁止词出现次数（赋能、助力、旨在、全方位等）
- 套话连接词（值得注意的是、需要指出的是等）
- 假大空句式（不仅...而且...、在这个...的时代等）

---

### 2. 结构合理性（20%）

**核心指标**：避免教科书结构

| 分数 | 标准 | 典型表现 |
|-----|------|---------|
| 9-10 | 结构新颖 | 逻辑流畅，过渡自然，有创意 |
| 7-8 | 结构清晰 | 有1-2处过渡生硬 |
| 5-6 | 结构尚可 | 有教科书痕迹，但不严重 |
| 3-4 | 结构死板 | 明显总分总，连接词堆砌 |
| 0-2 | 结构混乱 | 完全教科书式或逻辑混乱 |

**检测要点**：
- 是否为"总分总"或"并列三段论"
- 段落过渡是否靠连接词硬连
- 是否有"首先、其次、再次、最后"连续出现

---

### 3. 个人特色（20%）

**核心指标**：观点/情绪/经历表达

| 分数 | 标准 | 典型表现 |
|-----|------|---------|
| 9-10 | 特色鲜明 | 个人观点鲜明，情感真实，有独特洞见 |
| 7-8 | 有个人色彩 | 有3-5处个人表达，情感适度 |
| 5-6 | 略显客观 | 有1-2处个人表达 |
| 3-4 | 像说明书 | 几乎无个人色彩 |
| 0-2 | 完全客观 | 纯陈述，无任何个人特色 |

**检测要点**：
- "我觉得""我认为""说实话""坦白讲"等主观表达
- 个人经历描述（"我第一次用的时候..."）
- 情感词汇（惊讶、失望、兴奋、无语等）
- 吐槽或调侃

---

### 4. 句式规范（15%）

**核心指标**：长句控制，节奏感

| 分数 | 标准 | 典型表现 |
|-----|------|---------|
| 9-10 | 节奏感强 | 句式多变，长短交替，无超长句 |
| 7-8 | 整体规范 | 有1-2个超30字长句 |
| 5-6 | 需要优化 | 有3-5个超30字长句 |
| 3-4 | 节奏拖沓 | 长句较多，阅读费力 |
| 0-2 | 阅读困难 | 大量长句，需要重写 |

**检测要点**：
- 超过30字的句子数量
- 是否有单句成段制造节奏感
- 是否长短句交替

---

### 5. 风格一致性（10%）

**核心指标**：与选定风格的匹配度

| 分数 | 标准 | 典型表现 |
|-----|------|---------|
| 9-10 | 完全符合 | 标志性表达到位，风格鲜明 |
| 7-8 | 基本符合 | 偶有偏离，整体一致 |
| 5-6 | 部分符合 | 风格不够鲜明 |
| 3-4 | 风格混乱 | 多种风格混杂 |
| 0-2 | 完全不符 | 与选定风格无关 |

**检测要点**（以土著哥风格为例）：
- 是否使用01、02、03分段
- 是否有"说白了""什么意思？"等标志性表达
- 是否有金句收尾
- 是否以"我是土著哥，我们下期见"结尾

---

### 6. 可读性（10%）

**核心指标**：段落长度、过渡自然度

| 分数 | 标准 | 典型表现 |
|-----|------|---------|
| 9-10 | 一气呵成 | 段落适中，过渡自然 |
| 7-8 | 整体流畅 | 有1-2处卡顿 |
| 5-6 | 部分卡顿 | 段落过长或过渡生硬 |
| 3-4 | 体验差 | 多处卡顿，阅读费力 |
| 0-2 | 难以阅读 | 需要重写 |

**检测要点**：
- 段落长度是否适中（2-5句为宜）
- 段落之间是否有逻辑流
- 是否有突兀的跳跃

## 评分流程

### 自动评分步骤

```
1. 硬性条件检查
   ├── 通过 → 继续评分
   └── 不通过 → 标记问题，进入修改流程

2. 六维度评分
   ├── 语言自然度：检测禁止词、套话
   ├── 结构合理性：检测结构模式
   ├── 个人特色：检测主观表达
   ├── 句式规范：统计长句
   ├── 风格一致性：匹配风格特征
   └── 可读性：检测段落和过渡

3. 计算加权总分
   总分 = Σ(维度分数 × 权重)

4. 判断是否通过
   ├── 总分 ≥ 6.0 且 硬性条件全满足 → 通过
   └── 否则 → 进入修改流程
```

### 不通过时的修改流程

```
不通过 → 识别最低分维度 → 按优先级修改 → 重新评分
                                              ↓
                                    最多3轮自动修改
                                              ↓
                              ┌───────────────┴───────────────┐
                              ↓                               ↓
                            通过                          仍不通过
                              ↓                               ↓
                        输出给用户                    标记需人工审核
```

**修改优先级**：
- **P1**：硬性条件不满足（禁止词、长句、个人表达）
- **P2**：语言自然度 < 6分
- **P3**：结构合理性 < 6分
- **P4**：其他维度 < 6分

## 常见问题修改指南

### 禁止词替换

| 禁止词 | 替换为 |
|-------|-------|
| 赋能 | 帮助、让...更强 |
| 助力 | 帮、推动 |
| 旨在 | 就是为了、目的是 |
| 全方位 | 各方面、从头到尾 |
| 多维度 | 从几个角度 |
| 深度 | 仔细、认真 |
| 颠覆性 | 完全不一样的 |
| 值得注意的是 | （直接删除，进入正题） |
| 综上所述 | （直接删除） |

### 长句拆分示例

**原句**（52字）：
> 通过将先进的人工智能技术与传统的内容创作流程相结合，该工具能够显著提升创作者的工作效率，同时保持内容的高质量输出。

**拆分后**（3句，共32字）：
> 这工具把AI塞进了创作流程里。效率确实高了。质量也没掉。

### 增加个人表达示例

**原文**：
> 这个功能可以帮助用户快速完成任务。

**增加个人表达后**：
> 说实话，这个功能我第一次用的时候就惊了——真的能省不少时间。

## 与现有审核机制的整合

本评分系统与 `anti-ai-rules.md` 配合使用：

1. **anti-ai-rules.md**：提供具体的禁止词清单和替换策略
2. **quality-scoring.md**：提供量化评分标准和通过阈值

在第六阶段（毒舌主编审核）中：
1. 先按 anti-ai-rules.md 执行基础清洗
2. 再按 quality-scoring.md 进行量化评分
3. 不通过则自动修改并重新评分
