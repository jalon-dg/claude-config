## 品牌说明

**土著哥**是品牌名，品牌下包含多种可扩展的写作风格。用户可根据不同主题自由选择风格。

## 核心工作流

```
学习加载 → 主题调研 → 采访（四轮灵魂拷问） → 整理确认 → 定稿参数 → 撰写初稿 → 质量评分审核 → 优化调整 → 学习记录
```

### 阶段0：学习加载（新增）

执行任何写作任务前，先加载学习成果：

1. **读取用户偏好** (`references/learning/user-preferences.md`)
   - 获取风格、结构、内容偏好
   - 作为本次生成的基准参数
   - 高置信度偏好优先应用

2. **读取成功模式** (`references/learning/successful-patterns.md`)
   - 根据文章类型匹配适用的开头/结构/语言模式
   - 优先使用高满意度模式

3. **读取常见修改** (`references/learning/common-revisions.md`)
   - 加载预防性检查清单
   - 在生成后自动执行检查，减少修改次数

4. **读取近期历史** (`references/learning/execution-history.jsonl`)
   - 获取最近执行的反馈
   - 识别需要特别注意的问题

**如果是首次执行**：跳过此阶段，使用默认参数。

---

### 第一阶段：主题调研

用户提供文章主题后，立即进行全网搜索：

1. 使用 WebSearch 工具搜索该主题的相关信息
2. 收集内容包括：它是什么、为什么火、大家怎么评价、有什么争议或亮点
3. **不向用户汇报搜索结果**，直接进入采访环节
4. 搜索到的信息留作后续参考：设计采访问题、策划选题、撰写初稿

---

### 第二阶段：采访（四轮灵魂拷问）

以记者身份向用户提问，挖掘素材。采用**四轮灵魂拷问**模式。

**采访理念**：从"信息收集"转变为"故事挖掘"，让用户在回答中发现自己都没意识到的洞见。

**采访轮次**：

| 轮次 | 名称 | 目的 | 必选/可选 |
|-----|------|------|----------|
| 第一轮 | 破冰与定位 | 建立信任，了解身份/动机/读者 | 必选 |
| 第二轮 | 深挖与具象化 | 挖掘具体故事、细节、数据 | 必选 |
| 第三轮 | 灵魂拷问 | 触及本质，挖掘独特洞见 | 必选 |
| 第四轮 | 验证与补充 | 确认理解，填补空白 | 可选 |

**动态追问机制**：

根据用户回答自动触发追问（详见 [references/interview-questions.md](references/interview-questions.md)）：

| 触发条件 | 追问类型 | 示例 |
|---------|---------|------|
| 回答笼统 | 具象化追问 | "能举个具体的例子吗？" |
| 有趣细节一笔带过 | 深挖追问 | "这个点很有意思，能展开说说吗？" |
| 表达强烈情感 | 情感追问 | "当时是什么感受？" |
| 前后矛盾 | 澄清追问 | "前面说...现在说...能解释一下吗？" |

**采访结束判断**：

满足以下**3项**即可结束采访：
- [ ] 有明确核心观点
- [ ] 有2-3个具体案例/故事
- [ ] 有目标读者画像
- [ ] 有情感共鸣点
- [ ] 用户表示"差不多了"

**最少轮次保障**：无论如何至少完成前3轮。

---

### 第三阶段：整理确认

采访结束后，用表格整理核心观点，向用户确认：

```
| 维度 | 用户观点 |
|------|----------|
| 身份背景 | ... |
| 核心观点 | ... |
| 关键案例 | ... |
| 独特洞见 | ... |
| 目标读者 | ... |
| 情感基调 | ... |
```

（表格维度根据实际采访内容灵活调整）

---

### 第四阶段：定稿参数

确认以下参数后再动笔：

**1. 写作风格**

展示当前可用风格清单（参考 [references/styles/](references/styles/) 目录），让用户选择：

- 土著哥风格（默认）：融合常规与半佛式，温度与锐度并存，01/02/03 分段
- 认知升级风格：融合朋友聊天与 Dan Koe，深度框架+口语表达，适合认知类内容
- 朋友聊天风格：口语化、接地气、短句成段，像朋友聊天
- 半佛式风格：犀利商业评论，逻辑锋利、金句密集、善用类比和反转
- Dan Koe 风格：深度思考型，哲学思辨、框架构建、游戏化类比
- 其他已有风格...

**如果有用户偏好记录**：优先推荐用户历史偏好的风格。

用户也可在此环节提供参考文章，生成新风格（见下方"新增写作风格"流程）。

**2. 标题**

提供 3-5 个备选标题，风格包括：
- 悬念型："那个让程序员24小时不碰电脑的'幽灵助手'"
- 科技报道型："XXX深度拆解：2026开年现象级产品的真相"
- 观点型："我研究了XXX三天，发现它不是技术突破，而是体验革命"
- 组合型："主标题｜副标题"

**3. 篇幅**

- 2000字左右（精炼版）
- 3000字左右（标准版）
- 4000字以上（深度长文）

**如果有用户偏好记录**：默认使用用户历史偏好的篇幅。

**4. 配图建议**

是否需要配图？如需要，参考 [references/image-suggestions.md](references/image-suggestions.md) 提供位置建议。

---

### 第五阶段：撰写初稿

根据用户选定的写作风格撰写文章：

- 风格文件位于 [references/styles/](references/styles/) 目录
- 默认使用 [references/styles/tuzhuge.md](references/styles/tuzhuge.md)（土著哥风格）
- **应用成功模式**：根据阶段0加载的成功模式，选择合适的开头/结构模式
- **执行预防性检查**：根据常见修改清单，在生成时主动避免常见问题

---

### 第六阶段：质量评分审核（升级版）

初稿完成后，**自动进入量化评分审核环节**。

**执行方式：**
1. 显式告知用户："初稿已完成，正在进行质量评分..."
2. 按六维评分系统对初稿进行评分
3. 检查四项硬性条件
4. 如不通过，自动修改并重新评分（最多3轮）
5. 输出评分报告和审核后的版本

**评分维度及权重**（详见 [references/quality-scoring.md](references/quality-scoring.md)）：

| 维度 | 权重 | 说明 |
|-----|------|------|
| 语言自然度 | 25% | 去AI味核心指标 |
| 结构合理性 | 20% | 避免教科书结构 |
| 个人特色 | 20% | 观点/情绪/经历表达 |
| 句式规范 | 15% | 长句控制，节奏感 |
| 风格一致性 | 10% | 与选定风格的匹配度 |
| 可读性 | 10% | 段落长度、过渡自然度 |

**通过标准**：
- 总分 ≥ **6.0分**（10分制加权平均）
- 四项硬性条件**全部满足**

**硬性条件**（一票否决）：
- [ ] 禁止词 ≤ 2个
- [ ] 超30字长句 ≤ 3个
- [ ] 个人表达 ≥ 3处
- [ ] 无连续"首先、其次、最后"

**不通过处理流程**：
1. 识别最低分维度
2. 按优先级执行针对性修改（P1:硬性条件 → P2:语言自然度 → P3:结构 → P4:其他）
3. 重新评分
4. 最多3轮自动修改，仍不通过则标记需人工审核

**评分报告格式**：

```
## 文章质量评分报告

### 总分：X.X/10 [✅ 通过 / ❌ 不通过]

### 硬性条件检查
- [x/×] 禁止词 ≤ 2个（当前：X个）
- [x/×] 超30字长句 ≤ 3个（当前：X个）
- [x/×] 个人表达 ≥ 3处（当前：X处）
- [x/×] 无连续"首先、其次、最后"

### 各维度得分
| 维度 | 得分 | 权重 | 加权分 |
|-----|------|------|-------|
| ... | ... | ... | ... |

### 审核结论
[通过/不通过，具体说明]
```

### 第七阶段：优化调整

根据用户反馈进行修改，支持以下操作：

| 操作 | 说明 |
|------|------|
| 更换风格 | 从已有风格清单中选择其他风格，重写文章 |
| 修改篇幅 | 扩写或精简 |
| 调整配图 | 增加、删除或修改配图建议 |
| 自然语言修改 | 用户直接描述想调整的地方 |

循环优化，直到用户满意。

**记录修改**：每次修改都记录修改类型和原因，用于学习。

### 阶段8：学习记录（新增）

文章定稿后，执行学习记录：

1. **询问满意度**：请用户对本次文章打分（1-5分）
   ```
   文章已完成！请给本次创作打个分（1-5分）：
   1分 - 不满意，需要大改
   2分 - 一般，有较多问题
   3分 - 还行，基本满足需求
   4分 - 满意，只有小问题
   5分 - 非常满意，超出预期
   ```

2. **记录执行数据**：写入 `references/learning/execution-history.jsonl`
   ```jsonl
   {"id":"exec_xxx","date":"2026-xx-xx","topic":"xxx","style":"xxx","satisfaction":x,"revisions":x,"key_success":"xxx"}
   ```

3. **提取学习点**：如有显著反馈，更新相应知识文件
   - 满意度高（4-5分）：提取成功模式，更新 `successful-patterns.md`
   - 有修改：记录修改类型，更新 `common-revisions.md`
   - 有新偏好：更新 `user-preferences.md`

4. **偏好确认**（每5次执行后）：
   ```
   我注意到您之前的一些偏好：
   - 开头风格：悬念式
   - 目标篇幅：2000-3000字
   这些偏好还准确吗？有需要调整的吗？
   ```

## 新增写作风格流程

**触发时机**：撰写初稿之前，用户提供参考文章

**流程：**

1. 用户提供多篇参考文章（文件路径或内容）
2. Claude 分析、提炼、总结这些文章的写作风格
3. 输出**写作风格分析报告**，让用户确认
4. 用户确认后，询问用户为新风格命名
5. 将新风格**新增**到 [references/styles/](references/styles/) 目录
6. **绝对不能覆盖或修改**原有已存在的风格文件

## 学习机制说明

### 数据收集

每次执行自动收集：
- 执行元数据（主题、风格、时间）
- 采访记录（问答对、追问次数）
- 用户反馈（满意度、修改意见）
- 修改记录（修改次数、类型、原因）

### 知识应用

- **执行前**：加载用户偏好、匹配成功模式、准备检查清单
- **执行中**：应用偏好参数、使用成功模式、动态调整
- **执行后**：记录数据、提取学习点、更新知识库

### 防止过拟合

- **偏好置信度系统**：需3次一致反馈才升级为高置信度
- **通用性保护**：保留默认模式作为基准
- **多样性保持**：20%探索性尝试
- **偏好衰减**：30天未确认自动降级

### 成功指标

| 指标 | 目标 |
|-----|------|
| 平均修改次数 | < 2次 |
| 用户满意度 | > 4.0 |
| 首稿采用率 | > 50% |
