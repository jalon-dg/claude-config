| 当前置信度 | 降级条件 | 降级后 |

| 高 | 30天未确认 | 中 |
| 中 | 30天未确认 | 低 |
| 低 | 保持 | 低 |

### 偏好冲突处理

当新反馈与现有偏好冲突时：
1. 如果现有偏好置信度为"低"，直接更新
2. 如果现有偏好置信度为"中"，询问用户确认
3. 如果现有偏好置信度为"高"，记录冲突，下次执行时询问

## 偏好确认机制

### 定期确认

每5次执行后，主动与用户确认：

```
我注意到您之前的一些偏好：
- 开头风格：悬念式
- 目标篇幅：2000-3000字
- ...

这些偏好还准确吗？有需要调整的吗？
```

### 冲突确认

当检测到偏好冲突时：

```
我记得您之前偏好[旧偏好]，但这次您提到[新偏好]。
请问是：
1. 这次特殊，下次还是用[旧偏好]
2. 以后都用[新偏好]
```

## 默认偏好说明

以下是初始默认偏好的设计理由：

| 偏好项 | 默认值 | 理由 |
|-------|-------|------|
| 默认风格 | 土著哥风格 | 品牌主风格，温度与锐度并存 |
| 开头风格 | 悬念式 | 吸引力强，适合公众号传播 |
| 语言正式度 | 口语化(8/10) | 符合公众号阅读习惯 |
| 目标篇幅 | 2000-3000字 | 公众号标准长度 |
| 段落长度 | 2-4句 | 适合手机阅读 |

这些默认值会随着用户反馈逐步调整。
