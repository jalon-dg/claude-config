# 微信文章导出器 PC客户端 V1.0 需求说明与详细设计

## 一、项目背景

### 1.1 目标
基于现有的 `wechat-article-exporter` Web应用，开发一个Electron桌面客户端，保留核心功能：登录微信号、选择公众号、拉取文章、导出HTML、导出Markdown、导出电子书。

### 1.2 现有项目分析
- **技术栈**: Nuxt.js 3 + Vue 3 + TypeScript
- **核心功能**:
  - 微信公众号扫码登录 (`components/modal/Login.vue`)
  - 公众号搜索与选择 (`server/api/web/mp/*`)
  - 文章拉取 (`server/api/public/v1/article.get.ts`)
  - 导出功能 (`utils/download/Exporter.ts`, `composables/useExporter.ts`)
- **数据存储**: IndexedDB (Dexie)
- **部署模式**: 现有为Web应用，需要改为桌面客户端

## 二、需求说明

### 2.1 功能需求

| 功能模块 | 功能描述 | 优先级 |
|---------|---------|--------|
| 登录微信号 | 微信公众号扫码登录，支持二维码显示和状态轮询 | P0 |
| 选择公众号 | 搜索并选择目标公众号，展示公众号列表 | P0 |

| 导出Markdown | 将文章导出为Markdown格式 | P0 |
| 导出电子书 | 将文章导出为EPUB格式电子书 | P0 |

- Windows (主要)
- macOS (次要)

| 需求 | 描述 |

### 3.3 关键模块设计

#### 3.3.1 主进程入口 (electron/main.ts)
- 创建BrowserWindow
- 初始化系统托盘
- 注册原生菜单
- 设置IPC处理器
- 异常处理和日志记录

#### 3.3.2 预加载脚本 (electron/preload.ts)
- 暴露安全的API到渲染进程
- `electronAPI.saveFile()` - 文件保存对话框
- `electronAPI.readFile()` - 文件读取对话框
- `electronAPI.getAppPath()` - 获取应用路径
- `electronAPI.minimizeToTray()` - 最小化到托盘

#### 3.3.3 托盘服务 (electron/services/tray.ts)
- 创建系统托盘图标
- 托盘菜单项：显示窗口、退出
- 双击托盘图标显示窗口

#### 3.3.4 窗口管理 (electron/services/window.ts)
- 记录窗口位置和大小
- 恢复上次窗口状态
- 最小化到托盘功能

### 3.4 导出功能改造

现有Web应用使用 `FileSystemDirectoryHandle` 选择导出目录。Electron客户端改为：
1. 使用 `dialog.showSaveDialog()` 选择保存位置
2. 使用 `fs` 模块写入文件
3. 直接保存到用户指定目录，无需目录选择器

### 3.5 数据流

```
[微信扫码] --> [登录API] --> [存储Cookie] --> [获取公众号列表]
                                                        |
                                                        ▼
                                               [获取文章列表]
                                                        │
                                                        ▼
[选择导出格式] --> [调用Exporter] --> [保存到本地文件系统]
```
